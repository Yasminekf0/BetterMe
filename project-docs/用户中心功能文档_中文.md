# Master Trainer 用户中心功能文档

---

## 文档概述

本文档详细描述用户中心的所有功能模块、界面设计和操作流程，面向销售代表和解决方案架构师等普通用户。

---

## 1. 用户角色定义

### 1.1 目标用户

| 用户类型 | 描述 | 主要需求 |
|----------|------|----------|
| 销售代表 | 阿里云一线销售人员 | 提升销售技能、处理客户异议 |
| 解决方案架构师 | 技术解决方案专家 | 提升技术沟通能力、应对技术问题 |
| 合作伙伴销售 | 渠道合作伙伴销售人员 | 标准化产品知识、提升销售质量 |

### 1.2 用户权限

| 功能 | 权限 |
|------|------|
| 查看场景列表 | ✓ |
| 进行角色扮演练习 | ✓ |
| 查看个人反馈 | ✓ |
| 生成跟进邮件 | ✓ |
| 查看个人历史记录 | ✓ |
| 编辑个人资料 | ✓ |
| 查看他人练习记录 | ✗ |
| 管理场景 | ✗ |

## 3. 仪表板 (Dashboard)

### 3.1 功能描述
用户登录后的首页，提供个人练习概览、推荐场景和快速开始入口。

### 3.2 界面元素

#### 3.2.1 欢迎区域

```

│                                                                 │
│  👋 早上好，张三！                                               │
│                                                                 │
│  继续提升你的销售技能，今天已有 23 位同事完成了练习              │
│                                                                 │
│  [开始今日练习]                                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

#### 3.2.2 个人统计卡片

| 指标 | 描述 | 展示方式 |
|------|------|----------|
| 总练习次数 | 累计完成的练习数 | 数字 + 环比变化 |
| 平均得分 | 所有练习的平均分 | 数字 + 趋势箭头 |
| 本周练习 | 本周完成的练习数 | 数字 + 目标进度条 |
| 最佳得分 | 历史最高分 | 数字 + 场景名称 |

#### 3.2.3 进步趋势图

- 类型：折线图
- 数据：近10次练习的得分趋势
- 交互：悬停显示详情

#### 3.2.4 推荐场景

基于用户练习历史和薄弱环节，智能推荐适合的场景。

**推荐逻辑**：
1. 未练习过的场景
2. 得分较低需要改进的场景
3. 最近更新的新场景
4. 热门场景

#### 3.2.5 最近练习

显示最近3-5次练习记录，支持快速查看反馈或重新练习。

## 4. 场景中心 (Scenarios)

### 4.1 场景列表

#### 4.1.1 功能描述
展示所有可用的练习场景，支持搜索、筛选和排序。

#### 4.1.2 筛选条件

| 筛选项 | 选项 |
|--------|------|
| 分类 | 技术方案、异议处理、合规安全、竞争分析、开场白 |
| 难度 | 简单、中等、困难 |
| 状态 | 未练习、已练习、需改进 |
| 排序 | 推荐度、难度、练习次数、最新 |

#### 4.1.3 场景卡片信息

| 元素 | 描述 |
|------|------|
| 场景标题 | 场景名称 |
| 场景描述 | 简短描述（2行） |
| 难度标签 | 简单/中等/困难 |
| 分类标签 | 所属分类 |
| 预计时长 | 完成一次练习的预计时间 |
| 我的最佳分数 | 个人历史最高分（如有） |
| 平均得分 | 所有用户的平均分 |
| 练习次数 | 总练习次数 |

### 4.2 场景详情

#### 4.2.1 详情页面内容

**场景信息**
- 场景标题和详细描述
- 难度等级和预计时长
- 场景背景故事

**买家角色介绍**
- 角色头像和姓名
- 职位和公司
- 背景介绍
- 主要关注点
- 性格特征

**练习目标**
- 本场景的学习目标
- 需要掌握的技能点
- 评估维度说明

**我的历史记录**
- 历次练习得分
- 进步趋势
- 改进建议汇总

#### 5.2.2 功能元素

| 元素 | 功能 |
|------|------|
| 对话历史 | 显示完整对话记录，可滚动 |

| 发送按钮 | 发送消息 |
| 结束按钮 | 提前结束对话 |
| 帮助按钮 | 查看使用帮助 |

### 5.3 交互规则

#### 5.3.1 对话流程

1. **开场**：AI买家先发起对话
2. **多轮交互**：用户和AI轮流发言
3. **轮数限制**：默认最多8轮
4. **结束条件**：
   - 达到最大轮数
   - 用户主动结束
   - AI判断对话已完整

#### 5.3.2 输入验证

| 规则 | 说明 |
|------|------|
| 最小长度 | 至少10个字符 |
| 最大长度 | 最多2000个字符 |
| 空消息 | 不允许发送空消息 |
| 连续发送 | 等待AI回复后才能再次发送 |

#### 5.3.3 实时提示

基于对话内容，系统会提供实时提示：
- 买家提到的关键词
- 可能的回应策略
- 需要注意的要点
- 相关产品/功能信息

### 5.4 结束对话

#### 5.4.2 结束后流程

1. 显示加载动画："正在分析您的对话..."
2. 生成反馈报告
3. 自动跳转到反馈页面
